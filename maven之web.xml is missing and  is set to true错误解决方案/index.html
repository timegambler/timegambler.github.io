maven之web.xml is missing and is set to true错误解决方案

遇到这个问题一般是在导入其他项目发生的，路径不对造成的。
第一种网上大多提供的解决方案是：
1、双击tomcat

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/1_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

2、继续点击open lanch configuration

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/2_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

3、修改user Entries，主要是bootstrap.jar或者tomcat-juli.jar路径错误

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/3_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

第二种解决问题的方法：（ps：bootstrap.jar路径默认修改回/Servers/bin，按照上面一种方法修改跟本没用，会自动修改回来）
很简单，重新创建一个tomcat
1、先把已有的删掉，对着已有的tomcat右键，delete

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/4_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

2、点击create a new server

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/5_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

3、选择自己需要的version，然后next

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/6_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

4、点击download and install，选择下载路径，等待一会儿，finish按钮可用之后，就可以创建成功了。

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/7_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

![](/The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案/8_The archive: /Servers/bin/bootstrap.jar which is referenced by the classpath, does not exist.错误解决方案.png)

出现这个问题，表示没有找到web.xml
也有两种解决方案：
第一种是根据问题解决问题，非主流方式，但也能解决
（转载：https://www.cnblogs.com/dongyu666/p/7079414.html）
第二种是按照maven的配置，直接在properties中的project facets中勾选web项目

![](/maven之web.xml is missing and is set to true错误解决方案/1_maven之web.xml is missing and is set to true错误解决方案.png)
